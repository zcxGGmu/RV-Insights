完成当前项目的端到端测试（E2E），验证关键用户流程可用性并给出可追踪证据。

## 适用场景

- 发版前回归
- 关键流程改动后验证
- CI 失败后的本地复现与定位

## 执行步骤

1. 识别 E2E 测试框架与入口（优先 Playwright）。
2. 安装/校验依赖并启动必要服务（前后端、数据库、mock 服务）。
3. 运行核心旅程测试：
   - 登录/鉴权（若项目包含）
   - 主流程 happy path
   - 关键异常路径（网络失败、校验失败、权限不足等）
4. 收集产物：失败截图、视频、trace、测试报告。
5. 总结失败用例根因与修复建议。

## 最低输出要求

1. 测试环境说明（分支、命令、服务状态）
2. 用例结果汇总（通过/失败/跳过）
3. 失败详情（测试名 + 报错摘要 + 可能根因）
4. 产物位置（截图/视频/trace/report 路径）
5. 结论与下一步建议

## 质量门槛

- 不仅报告"命令通过"，还要覆盖关键业务路径。
- 若无法运行（环境缺失/服务异常），必须明确阻塞项与最小恢复步骤。
- 结论应能支持"是否可继续提测/发布"的判断。
